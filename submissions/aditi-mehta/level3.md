## What I Built

**The SMILE Gap Analyzer (Critic Agent)**

Instead of a basic chatbot that just answers the user question, I built the SMILE Gap Analyzer. It acts as a Senior Architect. When a user submits a digital twin idea, this agent audits the concept against the SMILE framework to find physical, human, and architectural blind spots.

I also engineered a **stateless IPC bridge** using Python's `subprocess` to solve `BrokenPipeError` issues, and added a Regex-based security layer to sanitize terminal inputs.

---

### Setup Instructions
You'll need Ollama (running `qwen2.5:1.5b`) and the LPI MCP Server built.
1. **Clone it:** `git clone https://github.com/Dynamic-ctrl/lpi-smile-agent`
2. **Setup Path:** Update `LPI_SERVER_PATH` in `agent.py` to point to your local `lpi-developer-kit/dist/src/index.js`.
3. **Run the Audit:** `python3 agent.py "I want to build a digital twin of an offshore wind farm"`

## Explainability

Every architectural critique generated by the agent contains an inline citation. Here is a sample of the actual terminal output:

```text
MISSING REALITY REPORT
==================================================
[SOURCE: LPI/smile_overview]: The power grid's high-voltage surges and potential for emergency blackouts necessitate a robust physical infrastructure to prevent, monitor, and manage these critical issues.

[SOURCE: LPI/get_case_studies]: The user's scenario highlights the importance of edge cases where the power grid is vulnerable to overloads or sudden failures due to environmental factors.

[SOURCE: LPI/smile_phase_detail]: In considering environmental variables, it's critical to include the digital twin system in real-time monitoring of weather forecasts and environmental data.
==================================================
```

The agent prints a full provenance summary at the end of every run:
```text
PROVENANCE - Every critique traced to its LPI source:
[1] Tool: smile_overview     -> Sourced baseline architectural safety hazards.
[2] Tool: query_knowledge    -> Sourced specific manual override constraints.
[3] Tool: get_case_studies   -> Sourced past failure metrics.
[4] Tool: smile_phase_detail -> Sourced sensor implementation gaps.
```
